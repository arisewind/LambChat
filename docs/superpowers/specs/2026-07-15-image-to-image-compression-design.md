# 图生图输入压缩设计

## 目标

图生图工具的 `input_images` 支持 URL、data URL 与沙箱文件路径；当读取到的图片超过 5 MiB 时，在发送给图像生成 API 前使用与聊天视觉输入相同的策略压缩：透明区域合成白底，编码为 JPEG（quality=82、optimize=True），且仅在压缩结果更小时替换输入。

## 设计

将图片压缩逻辑放入 `src/infra/image_utils.py`，返回 `(bytes, mime_type)`。它不修改原始上传文件或沙箱文件，并在 Pillow 解码或压缩失败时返回原始字节和 MIME 类型。`node_utils` 迁移为导入该函数，图生图工具也从该模块导入，避免 `infra` 依赖 `agents.core`。

聊天视觉输入、图片分析和图生图均调用该工具。图片分析继续经由 `inline_image_attachments_as_data_urls` 使用共享压缩器；图生图在 URL、data URL 或沙箱路径已经下载到临时文件后，通过 `run_blocking_io` 读取字节、调用共享工具并写入新的临时文件；若结果改变，则立即关闭原文件，并将 multipart 的文件名扩展名与 `content_type` 改为 JPEG。原有 20 MiB 下载上限和沙箱路径解析保持不变。

远程 URL 下载沿用现有图生图行为；本次不扩大网络访问范围，仅在已下载到临时文件的图像上执行压缩。data URL 和沙箱路径同样在下载完成后走该压缩步骤。

## 错误处理

压缩失败仅记录警告并继续使用原图；下载或解码错误维持现有错误行为。输入文件与压缩过程中创建的临时文件均由现有 `finally` 清理；成功替换时原文件已立即关闭，不会双重持有句柄。

## 测试

- 共享工具：超过阈值的带透明通道图片转为更小的 JPEG；未超过阈值保持不变。
- 图片分析：经 `inline_image_attachments_as_data_urls` 的图片仍使用同一压缩器。
- 图生图：沙箱路径、URL 和 data URL 输入超过阈值时，提交 multipart 的内容、MIME 类型和文件名均使用压缩后的 JPEG。
- 图生图：JPEG 未能变小或 Pillow 处理失败时，保留原始内容、MIME 类型与文件名，并继续请求。
- 生命周期：成功替换时原临时文件立即关闭；无论后续输入下载或 API 请求是否失败，替换后的文件均由 `_call_edit_api()` 的 `finally` 关闭。
- 透明度：测试解码压缩后的 JPEG，验证原先透明区域被合成为白色。
- 现有沙箱路径、下载大小上限和普通图生图请求测试继续通过。
